# Руководство по вкладу в проект

Спасибо, что хотите внести свой вклад в проект `portfolio-system-architect`! Этот документ описывает процесс внесения изменений в репозиторий.

## Как внести вклад

### 1. Форкните репозиторий

Создайте свой форк репозитория на GitHub.

### 2. Клонируйте репозиторий

```bash
git clone https://github.com/ваш-логин/portfolio-system-architect.git
cd portfolio-system-architect
```

### 3. Создайте ветку для вашей функции

```bash
git checkout -b feature/ваша-фича
```

### 4. Внесите изменения

Добавьте свои изменения в код, документацию или другие файлы.

### 5. Зафиксируйте изменения

```bash
git add .
git commit -m "Добавил: ваше описание изменений"
```

### 6. Отправьте изменения в ваш форк

```bash
git push origin feature/ваша-фича
```

### 7. Создайте Pull Request

Перейдите в оригинальный репозиторий и создайте Pull Request из вашей ветки.

## Стиль кодирования

- Следуйте общепринятым стандартам Python (PEP 8)
- Используйте типизацию (аннотации типов)
- Пишите понятные имена переменных и функций
- Добавляйте комментарии к сложным участкам кода

## Документация

- Обновляйте документацию при изменении функциональности
- Используйте Markdown для написания документации
- Добавляйте примеры использования

## Тестирование

- Добавляйте тесты для нового функционала
- Убедитесь, что все существующие тесты проходят

## Сообщения о коммитах

- Используйте повелительное наклонение: "Добавить", "Исправить", "Удалить"
- Пишите краткое, но информативное описание
- Указывайте номер задачи, если есть: "Исправить #123: ошибка в генерации сайта"

## Кодекс поведения

Пожалуйста, следуйте [Кодексу поведения](CODE_OF_CONDUCT.md) во всех взаимодействиях с сообществом.
