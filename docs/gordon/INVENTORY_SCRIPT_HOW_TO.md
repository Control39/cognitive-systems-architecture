# INVENTORY SCRIPT

## Что это

Автоматический скрипт на Python для сканирования всего диска C и создания структурированного отчёта о файлах.

Это нужно чтобы:
1. Найти ВСЕ твои проекты и файлы
2. Классифицировать их (код, документы, конфиги, данные)
3. Создать инвентарь для RAG-индексирования
4. Найти дубли и мусор

## Как использовать

### Шаг 1: Запустить скрипт

```bash
python C:\Users\Z\DeveloperEnvironment\projects\INVENTORY_SCRIPT.py
```

### Шаг 2: Дождаться результатов

Скрипт создаст папку `C:\Users\Z\DeveloperEnvironment\projects\INVENTORY_OUTPUT\` с файлами:

- `INVENTORY_REPORT.json` — для RAG (весь отчёт в JSON)
- `ALL_FILES.json` — список всех найденных файлов
- `INVENTORY_REPORT.txt` — читаемый текстовый отчёт

### Шаг 3: Посмотреть результаты

```bash
# Открыть текстовый отчёт (легче читать)
notepad C:\Users\Z\DeveloperEnvironment\projects\INVENTORY_OUTPUT\INVENTORY_REPORT.txt

# Или использовать JSON для обработки
```

## Что скрипт делает

```
Сканирует: C:\Users\Z (весь диск)
    ↓
Пропускает: .git, node_modules, __pycache__, .venv и т.д.
    ↓
Классифицирует файлы:
  - code (.py, .js, .ts, .go и т.д.)
  - docs (.md, .txt, .pdf и т.д.)
  - config (.json, .yaml, .env и т.д.)
  - scripts (.ps1, .sh, .bat и т.д.)
  - data (.csv, .xlsx, .sql и т.д.)
  - docker (Dockerfile, docker-compose.yml)
  - media (.png, .jpg, .mp4 и т.д.)
  - archive (.zip, .tar, .7z и т.д.)
  - other (остальное)
    ↓
Создаёт отчёт:
  - Статистика по типам
  - Список всех файлов
  - Найденные проекты (с Git или src)
```

## Зачем это нужно

### Для тебя сейчас

- Увидеть полную картину (что у тебя есть на диске)
- Понять структуру (какие проекты есть, где они)
- Найти дубли (одинаковые названия в разных местах)
- Подготовить данные для RAG-индексирования

### Для гранта / портфолио

Статистика:
- "Я обработала 5247 файлов и 20+ проектов"
- "Унифицировала 23 Docker файла в одну систему"
- "Проиндексировала 523 документа для RAG"

Это **доказательство** масштаба твоей работы.

### Для RAG-слоя

После инвентаризации можно:
1. Взять `ALL_FILES.json`
2. Прочитать все файлы типа "docs", "code", "config"
3. Отправить в RAG-индексер
4. Получить searchable knowledge base

## Если есть ошибки

Скрипт создаст список ошибок в отчёте. Обычные проблемы:

- Доступ запрещен (protected файлы) — игнорируются
- Очень длинные пути — берутся первые 250 символов
- Несуществующие файлы — пропускаются

Не критично, отчёт всё равно создастся.

## Дальше что?

После инвентаризации:

1. **Посмотри `INVENTORY_REPORT.txt`** — что ты нашла
2. **Проверь `ALL_FILES.json`** — есть ли неожиданные файлы / дубли
3. **Используй статистику в портфолио** — "5000+ файлов обработано"
4. **Подготовь файлы к RAG** — возьми все .md, .py, .json для индексирования

**Время запуска**: ~2-5 минут на 5000+ файлов  
**Требует**: Python 3.8+, 100MB свободного места
